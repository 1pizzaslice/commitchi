# Agent Instructions

This project is `commitchi`.

Before making changes, read these files in order:

1. `docs/HANDOFF.md`
2. `docs/ROADMAP.md`
3. `docs/PRD.md`
4. `docs/ARCHITECTURE.md`
5. `docs/DEV_HARNESS.md`

Important workflow rules:

- The user requested phased delivery. Stop at each phase boundary and wait for explicit approval before starting the next implementation phase.
- Current phase status is tracked in `docs/ROADMAP.md`.
- Keep product decisions in `docs/PRD.md` and implementation decisions in `docs/ARCHITECTURE.md`.
- Update `docs/HANDOFF.md` before ending a substantial work session.
- Do not run remote Git operations such as `git push` unless the user explicitly asks in that turn.
